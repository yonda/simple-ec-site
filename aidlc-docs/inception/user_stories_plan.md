### ステップ 2: アクター（利用者）の定義

- [x] システムを利用する主要なアクターをリストアップする
- [x] 各アクターの役割と目的を明確にする

**主要なアクター：**

1. **ゲストユーザー（未登録の訪問者）**
   - 役割：サイトを訪問し、商品を閲覧・購入する未登録ユーザー
   - 目的：書籍を検索・閲覧し、購入する

2. **登録済みユーザー**
   - 役割：アカウントを作成し、ログインしたユーザー
   - 目的：書籍を購入し、注文履歴を確認する

3. **システム**
   - 役割：自動的にビジネスルールを実行するバックエンドシステム
   - 目的：在庫管理を自動化し、データの整合性を保つ

[Question] ゲストユーザー（未登録ユーザー）の扱いについて教えてください：

- ゲストユーザーは商品閲覧のみ可能で、購入にはユーザー登録が必須ですか？
- それとも、ゲストユーザーも購入可能ですか？
- または、すべての機能でユーザー登録・ログインが必須ですか？

[Answer]
不要、ゲストユーザーも購入可能

[Question] 在庫管理について教えてください：

- 在庫管理は誰が行いますか？（購入時にシステムが自動的に在庫を減らす、など）
- 在庫がゼロになった商品の扱いはどうしますか？（表示しない、「在庫切れ」と表示して購入不可、など）

[Answer]
在庫管理は購入時にシステムが自動的に在庫を減らします。
在庫が ⁰ になった商品は在庫切れと表示し、購入は不可です

### ステップ 3: 主要なユースケースの抽出

- [x] 各アクターが実行する主要な操作を洗い出す
- [x] 優先度をつける（MVP に含めるもの、将来的に追加するもの）

**アクター別の主要な操作：**

**ゲストユーザー：**
- 商品一覧を閲覧する
- 商品を検索・フィルタリングする
- 商品詳細を閲覧する
- 商品をカートに追加する
- カート内容を確認・編集する
- 注文する（決済処理）
- ユーザー登録する
- ログインする

**登録済みユーザー：**
- ゲストユーザーのすべての操作
- 注文履歴を閲覧する
- ログアウトする

**システム：**
- 購入時に在庫を自動的に減らす
- 在庫切れ商品の購入を防ぐ
- 商品の在庫状態を表示する

**優先度（MVP に含めるもの）：**
- 必須（MVP）：
  - 商品一覧・詳細閲覧
  - 商品検索・フィルタリング
  - ショッピングカート（追加・編集・削除）
  - 注文・決済処理（モック）
  - 在庫管理（自動減算・在庫切れ表示）
  - ユーザー登録・ログイン（任意）
  - 注文履歴（登録済みユーザーのみ）
